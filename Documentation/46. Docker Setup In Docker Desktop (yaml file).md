# Docker & Docker Compose Documentation for Restaurants Project

This documentation explains how to build and run the ASP.NET 8 backend and React frontend using Docker and Docker Compose.

## ✅ Architecture Overview

This project uses a modern cloud-based architecture:

```conf
Client Browser
      ↓
React SPA (Docker + Nginx)
      ↓
ASP.NET 8 Web API (Docker)
      ↓
Supabase Managed PostgreSQL (Cloud)
```

---

🚀 Tech Stack

```conf
Frontend   : React + Vite + Nginx
Backend    : ASP.NET 8 Web API
Database   : Supabase PostgreSQL (Managed Cloud)
Container  : Docker + Docker Compose
```

## 1️⃣ Docker Compose Overview (docker-compose.yml)

We use Docker Compose to orchestrate multiple services:

```yaml
version: "3.9"

services:
  # ---------------------------
  # ASP.NET 8 Backend
  # ---------------------------
  backend:
    container_name: restaurants_backend
    build:
      context: . # VERY IMPORTANT
      dockerfile: ./Rider/Restaurants.Api/Restaurants.Api/Dockerfile
    ports:
      - "5050:80"
    environment:
      ASPNETCORE_ENVIRONMENT: Development
      ConnectionStrings__DefaultConnection: ${DB_CONNECTION}

  # ---------------------------
  # React Vite Frontend
  # ---------------------------
  frontend:
    container_name: react_frontend
    build:
      context: ./React/restaurant-react
      dockerfile: Dockerfile
    ports:
      - "3000:80"
    depends_on:
      - backend
    environment:
      VITE_API_URL: "http://backend:80" # 👉 The frontend calls the backend
```

Key Notes:

- context and dockerfile paths must be correct relative paths.
- depends_on ensures services start in the correct order.
- Environment variables define connection strings and API URLs for Docker containers.

---

## Environment Variables (Highly Recommended)

Never hardcode secrets inside docker-compose.yml.

Create a .env file at the project root:

### File: .env

In main folder same level with docker-compose.yaml create .env file

```yaml
DB_CONNECTION=Host=aws-xxxx.pooler.supabase.com;Port=5432;Database=postgres;Username=your-user;Password=your-password;SSL Mode=Require;Trust Server Certificate=true;
```

Benefits:

- Prevent credential leaks
- Safe for Git repositories
- Industry best practice
- Required in most production environments

## 2️⃣ Backend Dockerfile (Rider/Restaurants.Api/Restaurants.Api/Dockerfile)

```yaml
# Stage 1: Build
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src

# Copy only project files first (to cache dependencies)
COPY Rider/Restaurants.Api/Restaurants.Api/*.csproj Restaurants.Api/
COPY Rider/Restaurants.Api/Restaurants.Application/*.csproj Restaurants.Application/
COPY Rider/Restaurants.Api/Restaurants.Domain/*.csproj Restaurants.Domain/
COPY Rider/Restaurants.Api/Restaurants.Infrastructure/*.csproj Restaurants.Infrastructure/

RUN dotnet restore Restaurants.Api/Restaurants.Api.csproj

# Copy all other files
COPY . .

# Publish app
RUN dotnet publish Rider/Restaurants.Api/Restaurants.Api/Restaurants.Api.csproj -c Release -o /app/publish

# Stage 2: Runtime
FROM mcr.microsoft.com/dotnet/aspnet:8.0
WORKDIR /app
COPY --from=build /app/publish .

ENTRYPOINT ["dotnet", "Restaurants.Api.dll"]

```

Important Notes:

- Copy paths (COPY) must match your project folder structure exactly.
- Separate build stage and runtime stage for smaller image size.
- Always use dotnet restore before copying all files for caching dependencies.

## 4️⃣ Backend Connection String

Make sure connection string in docker-compose.yml matches your backend:

```yaml
DB_CONNECTION=Host=aws-xxxx.pooler.supabase.com;Port=5432;Database=postgres;Username=your-user;Password=your-password;SSL Mode=Require;Trust Server Certificate=true;
```

✅ Backend Connection (Supabase)

This project uses a managed PostgreSQL database hosted in the cloud, so no database container is required.

Important notes:

- The backend connects directly to Supabase via SSL.
- Docker networking is NOT used for the database.
- The database is accessible securely over the internet.

## 3️⃣ React Dockerfile (React/restaurant-react/Dockerfile)

```yaml

# Stage 1: Build
FROM node:20-alpine AS build
WORKDIR /app

# Install dependencies
COPY package.json package-lock.json* ./
RUN npm install

# Copy source code and build
COPY . ./
RUN npm run build

# Stage 2: Serve with Nginx
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf


EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]


```

Important Notes:

- Two-stage build: Node to build, Nginx to serve.
- Ensure paths are correct. The dist folder must exist after npm run build.

## 5️⃣ React Axios Configuration

Update axios.ts to use Docker environment variable:

```ts
// Read API base URL from environment
const apiBaseURL = import.meta.env.VITE_API_URL;

const api = axios.create({
  baseURL: `${apiBaseURL}/api`, // Backend base URL for Docker

  // baseURL: "http://localhost:5204",  // For local testing outside Docker
});
```

VITE_API_URL is set in docker-compose.yml for Docker runtime.

During local development, you can uncomment the localhost line for testing.

## Nginx Configuration

### File: nginx.conf

```conf
server {
    listen 80;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        # For React Router SPA support
        try_files $uri /index.html;
    }

    location /assets/ {
        try_files $uri =404;
    }

    location = /favicon.ico {
        try_files $uri =404;
    }
}
```

**Why nginx.conf Is Required**

When using React Router, routes like:

```yaml
/dashboard
/profile
/orders
```

do not exist as physical files.

Without nginx configuration:

👉 Refresh → 404 Not Found

The rule in `nginx.conf` below forces all requests to return index.html, allowing React Router to handle navigation.

```conf
try_files $uri /index.html;
```

## 6️⃣ Kestrel Port Configuration for Docker and Localhost

In order for the ASP.NET backend to work both inside a Docker container and during local development, you need to explicitly configure Kestrel to listen on multiple ports.

Add the following before any service registration in `Program.cs`:

```csharp
// ADD THIS BEFORE ANY OTHER SERVICE REGISTRATION FOR DOCKER PORT BINDING
builder.WebHost.ConfigureKestrel(options =>
{
    // Docker container port
    options.ListenAnyIP(80);

    // Local development ports (optional)
    options.ListenLocalhost(5000);   // HTTP
    options.ListenLocalhost(5001);   // HTTPS (optional)
});
```

Key Notes:

- ListenAnyIP(80) ensures the container can accept requests from the Docker network (other services like React frontend).
- ListenLocalhost(5000/5001) keeps local development working without Docker.
- Place this before builder.Services.Add... to avoid conflicts with service registration.

## 7️⃣ Running the Docker Environment

After creating your docker-compose.yml and Dockerfiles:

1. Build the Docker images

From the root of your project (where docker-compose.yml is located):

```yaml
docker-compose up --build
```

This will:

- Build the backend using your ASP.NET Dockerfile
- Build the frontend using your React Dockerfile
- Pull the PostgreSQL image if not already available

2. Start all services

```yaml
docker-compose up
```
