# Restaurants Project EC2 Deployment Guide

This guide will walk you through setting up an EC2 server, configuring Podman, preparing backend and frontend environment variables, building Docker images locally, uploading them to EC2, and running the containers in a clean order.

## 4️⃣ Configure Backend CORS (Mandatory)

### File: program.cs
```csharp
// Read frontend URL from environment variable (backend.env)
var frontendUrl = builder.Configuration["FRONTEND_URL"];

// Configure CORS using the env variable
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowReact", policy =>
    {
        policy.WithOrigins(frontendUrl ?? "http://localhost:3000")
              .AllowAnyHeader()
              .AllowAnyMethod();
    });
});

app.UseCors("AllowReact");
```

## 5️⃣ Configure Axios in Frontend

```ts
import axios from "axios";
const apiBaseURL = import.meta.env.VITE_API_URL; //Read From frontend.env file
const api = axios.create({
  baseURL: `${apiBaseURL}/api`,
});
export default api;
```

## 7️⃣ Build Docker Images Locally

```bash
# Build Docker Image Using AMD64 Format
docker buildx build --platform linux/amd64 -t restaurants-backend-amd64 ./Rider/Restaurants.Api/Restaurants.Api
docker buildx build --platform linux/amd64 -t restaurants-frontend-amd64 ./React/restaurant-react
# Check Docker Images
docker images
# Save Images in Tar Format
docker save -o restaurants-backend-amd64.tar restaurants-backend-amd64:latest
docker save -o restaurants-frontend-amd64.tar restaurants-frontend-amd64:latest
```

---

## 8️⃣ Upload Docker Images to EC2

Upload Both Images (frontend + backend)
```bash
scp -i icrest-abinitio-training.pem restaurants-backend-amd64.tar engineer@ec2-43-216-59-160.ap-southeast-5.compute.amazonaws.com:/home/engineer/
scp -i icrest-abinitio-training.pem restaurants-frontend-amd64.tar engineer@ec2-43-216-59-160.ap-southeast-5.compute.amazonaws.com:/home/engineer/
```

---

## 9️⃣ Load Images on EC2

Load the image instead of using tar file
```bash
podman load -i restaurants-backend-amd64.tar
podman load -i restaurants-frontend-amd64.tar
```

---

## 🔟 Run Containers (Clean Start)

### Backend

```bash
podman stop backend || true
podman rm backend || true
podman run -d --name backend -p 5050:80 --env-file ./backend.env docker.io/library/restaurants-backend-amd64:latest
```

### Frontend

```bash
podman stop frontend || true
podman rm frontend || true
podman run -d --name frontend -p 3000:80 --env-file ./frontend.env docker.io/library/restaurants-frontend-amd64:latest
```

✅ **Workflow Summary**
1. Add `engineer` user & configure SSH
2. Install Podman & optional podman-compose
3. Configure backend CORS (**mandatory**)
4. Configure Axios to read frontend environment variable
5. Prepare `.env` files **before building Docker images**
6. Build Docker images locally (backend & frontend)
7. Save images and upload to EC2
8. Load images on EC2
9. Stop old containers & run new ones with `.env`
10. Verify logs and endpoints
