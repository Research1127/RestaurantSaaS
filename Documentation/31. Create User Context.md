# User Context Setup 

**User Context**: A framework-independent way to access the currently authenticated user and their roles in the Application layer, used to decouple business logic from ASP.NET and support testable, role-based authorization.

This document explains how the **User Context** is implemented in the Application layer to provide a clean, framework-independent way to access authenticated user information.

---


## Create CurrentUser

The `CurrentUser` record represents the authenticated user and their roles.

### File: `CurrentUser.cs`
Create it in our Application Module under `User/CurrentUser.cs`

```csharp
namespace Restaurants.Application.User;

public record CurrentUser(string Id, string Email, IEnumerable<string> Roles)
{
    public bool IsInRole(string role) => Roles.Contains(role);
}
```
Purpose
- Holds basic user information: Id, Email, and Roles
- Provides a helper method IsInRole to check user roles
- Immutable by default (using record)
- Used in the Application layer to decouple business logic from ASP.NET

## UserContext Implementation

The UserContext class implements IUserContext and reads user information from the HTTP context.

### File: UserContext.cs

```csharp
public class UserContext(IHttpContextAccessor httpContextAccessor) : IUserContext
{
    public CurrentUser? GetCurrentUser()
    {
        var user = httpContextAccessor?.HttpContext?.User;
        if (user == null)
        {
            throw new InvalidOperationException("User Context is not present");
        }

        if (user.Identity == null || !user.Identity.IsAuthenticated)
        {
            return null;
        }
        
        var userId = user.FindFirst(c => c.Type == ClaimTypes.NameIdentifier)!.Value;
        var email = user.FindFirst(c => c.Type == ClaimTypes.Email)!.Value;
        var roles = user.Claims.Where(c => c.Type == ClaimTypes.Role)!.Select(c => c.Value);
        
        return new CurrentUser(userId, email, roles);
    }
}
```
Key Points
- Reads the current authenticated user from HttpContext
- Throws an exception if the HTTP context is missing
- Returns null if the user is unauthenticated
- Extracts:
  1. Id from ClaimTypes.NameIdentifier 
  2. Email from ClaimTypes.Email 
  3. Roles from ClaimTypes.Role 
- Returns a CurrentUser object

---

## IUserContext Interface

The interface defines a contract for accessing the current user.

Create it in the same file as UserContext and apply the member there.

```csharp
public interface IUserContext
{
    CurrentUser? GetCurrentUser();
}
```
---

## Dependency Injection

Register IUserContext and HttpContextAccessor in the service collection.

### File: ServiceCollectionExtensions.cs

Add in method `AddApplicaton`

```csharp
services.AddScoped<IUserContext, UserContext>();
services.AddHttpContextAccessor();
```

Purpose
- Allows controllers and services to inject IUserContext
- Provides access to the current user in the Application layer
- Supports testing by allowing mocking of IUserContext
